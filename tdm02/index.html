    Exercice 1 : afficher une fenêtre de couleur en JAVA¶

    Faites fonctionner le programme suivant : 

    package fr.esisar;

import java.awt.Color;

import javax.swing.JFrame;

public class ColorFrame
{
    public static void main(String[] args) throws Exception
    {
        ColorFrame c = new ColorFrame();
        c.execute();
    }

    private void execute() throws Exception
    {
        JFrame frame = new JFrame("Chenillard");
        frame.setSize(300,300);

        //
        frame.getContentPane().setBackground(Color.GREEN);
        frame.setVisible(true);
        Thread.sleep(2000);


        //
        frame.getContentPane().setBackground(Color.RED);
        frame.setVisible(true);
        Thread.sleep(2000);


        frame.getContentPane().setBackground(Color.GREEN);
        frame.setVisible(true);
        Thread.sleep(2000);

        frame.dispose();

    }
}

    Que fait il ? Analyser son fonctionnement interne.

    Exercice 2 : réaliser une "télécommande" avec netcat¶

    Réaliser maintenant un programme Television.java qui affiche au départ une fenêtre verte.

    Ce programme doit être pilotable à distance avec netcat.

    La fenêtre verte doit devenir rouge si on envoie la commande "red" avec netcat : 

    netcat -u 127.0.0.1 7050 
red

    La fenêtre doit devenir verte si on envoie la commande "green" avec netcat : 

    netcat -u 127.0.0.1 7050 
green

    Ensuite, au lieu de piloter votre programme local, pilotez un programme distant (sur une autre machine)

    Exercice 3 : Le ping pong UDP¶

    L'objectif de cet exercice est de créer l'effet visuel ci dessous avec deux programmes.

    Pour faire cela, vous devez faire deux programmes Java (un pour chaque fenêtre).

    Le programme 1 écoute sur le port 4001, le programme 2 écoute sur le port 4002 (en UDP toujours).

    Le fonctionnement du programme 1 est le suivant :

    • il passe rouge
    • il attend une seconde
    • il passe vert
    • il envoie un message au programme 2 pour lui dire de passer rouge
    • il attend du programme 2 l'ordre de repasser en rouge
    • et ainsi de suite

    Le fonctionnement du programme 2 est le suivant :

    • il est vert
    • il attend du programme 1 l'ordre de passer en rouge
    • il passe rouge
    • il attend une seconde
    • il passe vert
    • il envoie un message au programme 1 pour lui dire de passer rouge
    • et ainsi de suite

    On peut considérer que les deux programmes font en quelque sorte une partie de ping pong, en s'échangeant une balle qui est un message UDP.

    Exercice 5 : Le chenillard UDP¶

    L'objectif de cet exercice est de créer l'effet visuel ci dessous avec 4 instances d'un même programme (c'est à dire un programme que vous lancerez 4 fois).

    La communication entre les instances du programme doit se faire en UDP.

    Vous devez faire un seul programme, que vous lancerez 4 fois, avec des paramètres différents sur la ligne de commande.

    Exercice 6 : Le chenillard UDP auto adaptatif¶

    Dans l'exercice précédent, vous avez noté une certaine lourdeur dans l'utilisation de votre chenillard : impossibilité de rajouter facilement un cinquième écran, si l'écran 3 s'arrête, alors tout le système s'arrête, …

    Dans cet exercice, vous allez réalisez deux programmes :

    • un programme serveur ordonnanceur
    • un programme client

    Le programme client sera lancé N fois et correspond à un terminal clignotant, le programme serveur sera lancé une seule fois. Le fonctionnement du programme client sera le suivant :

    • démarrage du programme
    • il s'enregistre auprès du serveur
    • il attend ensuite les ordres du serveur : un ordre pour passer en rouge, un ordre pour passer en vert

    Le fonctionnement du programme serveur sera le suivant :

    • il gère l'enregistrement des clients les uns après les autres
    • le dernier client se connecte (pour cela, un paramètre sur la ligne de commande permet d'indiquer au programme client qu'il est le dernier, le dernier client s'enregistre avec un message spécifique)
    • le programme serveur envoie ensuite les ordres aux clients pour gérer correctement l'affichage des couleurs
