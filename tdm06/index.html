    TDM06¶

    Le TDM 06 va permettre d'appronfondir la gestion des threads, en Java.

    Exercice 1 : Le problème de la concurrence.¶

    Executez le code ci dessous :

    package tdm.tdm08;

public class Calculatrice extends Thread
{
    private Somme somme;

    public Calculatrice(Somme somme)
    {
        this.somme = somme;
    }

    @Override
    public void run()
    {
        int res = 0;
        for (int i = 0; i < 1000; i++)
        {
            res= somme.somme(res, i);
        }
        System.out.println("La somme 1+2+3+4+...+998+999 est égale à :"+res);
    }


    public static void main(String[] args) throws InterruptedException
    {
        Somme somme = new Somme();
        Calculatrice c1 = new Calculatrice(somme);
        Calculatrice c2 = new Calculatrice(somme);

        c1.start();
        c2.start();
    }


    static  class Somme
    {

        int c;

        public int somme(int a, int b) 
        {
            c=a+b;
            System.out.println("c="+c);
            return c;
        }

    }

}

    Que fait ce code ? Les résultats affichés sont ils justes ?

    Corriger ce code en utilisant les deux méthodes vues en cours.

    Exercice 2 : Somme TCP (20 points)¶

    Présentation générale du problème.¶

    Un serveur TCP est lancé sur une machine, ce serveur écoute en TCP sur tous les ports compris entre 21 000 et 23 000 (inclus).

    Ce serveur fonctionne ainsi :

    • le serveur écoute sur tous les ports compris entre 21 000 et 23 000 (inclus)
    • quand un client se connecte sur un de ces ports, le serveur attend un message contenant le texte COMBIEN
    • quand le serveur reçu ce message COMBIEN, le serveur répond environ 5 à 7 secondes plus tard
    • la réponse du serveur est MONTANT=15EURO
    • les textes MONTANT= et EURO sont fixes, par contre la valeur du montant (ici 15) peut varier entre 1 et 10000.
    • une fois la réponse envoyée, le serveur ferme la socket de connexion.

    Le serveur répond « VOUS AVEZ FAIT UNE ERREUR.» si le message envoyé par le client est différent de COMBIEN.

    L'objectif de cet exercice est de réaliser un programme capable de déterminer :

    • le port X correspondant au montant maximum
    • la valeur du montant maximum
    • la somme des valeurs retournées par chaque port

    Le programme devra afficher ces 3 valeurs.

    Préparation de votre machine.¶

    Pour pouvoir tester votre programme, vous allez installer sur votre machine ce serveur TCP. Pour cela, télécharger le fichier somme-tcp.jar

    somme-tcp.jar

    Lancer ce serveur avec la commande 

    java -jar  somme-tcp.jar

    Ecriture du programme¶

    Ecrivez maintenant une classe SommeTCP. Si on exécute le main de cette classe, on obtient l'affichage suivant :

    Début de la recherche ...
Le montant maximum est 8500 euros
Le port d'écoute correspondant à ce maximum est 21500
La somme des montants retournés par tous les ports est 9145123
Fin du programme

    Cet affichage devra se faire dans un délai inférieur à 1 minute.
